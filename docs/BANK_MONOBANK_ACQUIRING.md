# Monobank: еквайринг (Acquiring) — інструкція для проєкту P-3-0

Документ фіксує **що вже зроблено**, **як це працює** і **на що звертати увагу**, коли повертаємось до теми інтеграції еквайрингу (детальна виписка по оплатах vs агреговані рядки в Personal API).

---

## 1) Два різні світи Monobank

| Що | Personal API | Acquiring API |
|----|----------------|---------------|
| Токен | З `api.monobank.ua` (підключення рахунків у нас) | Окремий токен з кабінету еквайрингу / мерчанта (див. [документацію](https://api.monobank.ua/docs/acquiring.html)) |
| Виписка по рахунку | `GET /personal/statement/{account}/{from}/{to}` — рух коштів по картці/рахунку | `GET /api/merchant/statement?from=&to=` — **окремі** еквайринг-транзакції (invoice, maskedPan, amount, profitAmount тощо) |
| Агреговані нарахування | Так (наприклад «Покриття за проведені трансакції…») — **без** розбиття по клієнтах | Тут якраз деталізація по платежах мерчанта |

**Важливо:** токен, який ми зберігаємо в `BankConnection` при «Підключенні рахунків», призначений для **Personal API**. Якщо тим самим токеном викликати `GET /api/merchant/statement`, Monobank може повернути **порожній `list: []`** або помилку — це очікувана ситуація, якщо токен не є **еквайринговим**.

---

## 2) Що вже реалізовано в коді

### Backend

- **Файл:** `web/app/api/bank/acquiring/statement/route.ts`
- **Наш endpoint (потрібен доступ до розділу Банк):**  
  `GET https://p-3-0.vercel.app/api/bank/acquiring/statement`
- **Зовнішній виклик:**  
  `GET https://api.monobank.ua/api/merchant/statement?from=<unix>&to=<unix>`  
  Заголовок: `X-Token: <токен з обраного підключення>`

**Логіка вибору підключення:**

1. Шукається підключення monobank з назвою, що містить **«Жалівців Олександра»** (точне або часткове співпадіння).
2. Якщо не знайдено — береться **перше** підключення monobank, у якого є рахунок з потрібним IBAN (див. нижче).
3. Рахунок визначається за **повним IBAN** `UA203220010000026000360049085` (нормалізація пробілів/регістру), інакше за останні 4 цифри **9085** у `iban` / `maskedPan` / `externalId`.

**Період виписки (на момент останнього оновлення документа):** з **01.03.2026** по **17.03.2026** (UTC, межі дня).

**Відповідь API для UI:** `ok`, `dateFrom`, `dateTo`, `connectionName`, `accountHint`, `endpoint`, `summary` (кількість транзакцій, збіг по картці …9085, суми), `filteredItems`, санітизований `raw`. Чутливі поля в `raw` маскуються перед віддачею на клієнт.

### Frontend (Банк 1)

- **Файл:** `web/app/(admin)/admin/bank/connections/page.tsx`
- Сторінка: `/admin/bank/connections` (кнопка **Банк 1** з `/admin/bank`).
- Блок **«Acquiring (01.03–17.03.2026, рахунок …9085)»** розміщений **перед** секцією **«Виписка»** (Personal).
- Кнопка викликає наш `GET /api/bank/acquiring/statement` і показує таблицю + розкривний raw JSON.

### Що **не** зроблено (для наступних ітерацій)

- Окреме поле в БД для **токена еквайрингу** (не змішувати з Personal-токеном у проді без розуміння ризиків).
- Автоматичне зіставлення агрегованих рядків Personal з рядками Acquiring (CRM/облік).
- Налаштовувані з UI дати періоду та вибір підключення/мерчанта.

---

## 3) Чому в інтерфейсі «0 транзакцій»

Можливі причини:

1. **Токен Personal, а не Acquiring** — найімовірніше для поточної схеми зберігання токена.
2. За обраний період **немає** еквайринг-операцій по цьому мерчанту.
3. Токен прив’язаний до **іншого** мерчанта (тест/інший ФОП).

**Що зробити далі:** отримати в кабінеті Monobank **токен саме для еквайрингу** і або додати окреме підключення в проєкті під нього, або тимчасово підставити в тестовий endpoint / env — після цього `list` має заповнитись при наявності операцій.

---

## 4) Корисні посилання

- Acquiring API (загальне): https://api.monobank.ua/docs/acquiring.html  
- Виписка за період: `GET /api/merchant/statement` (параметри `from`, `to`, опційно `code` для субмерчанту).
- Існуюча інструкція Personal + webhook: `web/MONOBANK_API_WEBHOOK_SETUP.md`

## 5) Швидкий чеклист для AI / розробника при поверненні до задачі

- [ ] Підтвердити, який токен використовується: Personal vs Acquiring.
- [ ] Якщо потрібна реальна виписка — передбачити зберігання **окремого** acquiring-токена (Prisma + міграція).
- [ ] Константи періоду та IBAN зараз **зашиті** в `route.ts` — винести в конфіг/UI за потреби.
- [ ] Фільтр `…9085` стосується **maskedPan** у відповіді еквайрингу, не IBAN рахунку ФОП.
