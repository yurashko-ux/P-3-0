# Посвідчення про відрядження: PDF з готового шаблону (друк)

Цей документ описує **повторювану дію**: з **PDF-шаблону** (готовий макет друку, зазвичай дві форми на аркуші) і **витягу з наказу** зібрати **PDF для друку** з підставленими реквізитами **без зсуву** відносно ліній шаблону.

## Безпека та GitHub

**У репозиторій не потрапляють і не повинні комітитися:**

- ПІБ, військові звання, номери наказів/розпоряджень, дати з реальних документів;
- шляхи до домашніх каталогів користувача;
- згенеровані PDF з персональними даними;
- файл із підстановками `vidryadzhennya.replacements.json` (реальні значення).

У git лежать лише **опис процесу**, **скрипт** і **приклад структури** (`*.example.json`) без робочих даних. Реальні пари рядків `old` / `new` зберігаються **локально** (див. `.gitignore`).

## Що передає користувач (поза git)

1. **Шаблон** — PDF з попереднього тиражу з тим самим макетом, що піде в друк.
2. **Дані** — витяг з наказу (наприклад `.doc`) або текст із новими реквізитами: дата виїзду, номер/дата розпорядження, перший ПІБ у бланку, якщо він відрізняється від першого рядка в шаблоні.

## Куди зберігати результат

Локально на машині користувача, у зручній папці (наприклад каталог для відряджень на робочому столі). **Шлях не фіксується в репозиторії** — передається як `--output`.

## Скрипт у репозиторії

Файл: [`scripts/vidryadzhennya_from_pdf_template.py`](../scripts/vidryadzhennya_from_pdf_template.py)

Залежність: `pip install pymupdf`

Підстановки беруться з **локального JSON** (не з коду), наприклад:

```bash
python3 scripts/vidryadzhennya_from_pdf_template.py \
  --template /шлях/до/шаблону.pdf \
  --output /шлях/до/результату_друк.pdf \
  --replacements /шлях/поза/репо/vidryadzhennya.replacements.json
```

Шрифти за замовчуванням (macOS): Times New Roman з `System/Library/Fonts/Supplemental/`; можна `--font-regular` та `--font-bold`.

Структура JSON — див. [`scripts/vidryadzhennya.replacements.example.json`](../scripts/vidryadzhennya.replacements.example.json) (скопіювати без суфікса `.example`, заповнити, **не комітити**). Ключ `NAME_FIRST` (старий/новий ПІБ у першому посвідченні) **необов’язковий** — якщо його немає, заміна ПІБ на першій сторінці не виконується.

## Як отримати точні рядки `old`

Рядки `old` мають **один в один** збігатися з текстом у шрифтовому шарі PDF (у т.ч. лапки U+201C/U+201D навколо `__`, пробіли на кінці рядка). Зняти їх через PyMuPDF: `page.get_text("dict")` → поле `span["text"]`.

## Технічні правила (щоб текст не «їхав»)

1. Шрифт як у шаблоні: **Times New Roman** (звичайний і **Bold** для рядка ПІБ після «Видано», якщо так у PDF).
2. Розмір — з поля `size` того ж `span`, що замінюється.
3. Позиція — після редукції вставка в **`origin` цього span** з **`rotate=90`**; не використовувати лише кут `search_for` як єдину опору.
4. Редукція по повному `bbox` span.
5. Порожню останню сторінку шаблону за наявності — видалити перед збереженням.

## Чеклист для агента

1. Отримати шляхи до **шаблону PDF** і **витягу** (текст наказу) — **не** дублювати зміст витягу в файлах репозиторію.
2. Заповнити **локальний** `vidryadzhennya.replacements.json` (за зразком `.example.json`).
3. Запустити скрипт з `--template`, `--output`, `--replacements`.
4. Перевірити 1–2 сторінки результату (дві форми на аркуші, вирівнювання, шрифт).
