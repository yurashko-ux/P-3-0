# Відрядження з PDF-шаблону: правила верстки та винятки

Документ доповнює [`VIDRYADZHENNYA_PDF_Z_SHABLONU.md`](VIDRYADZHENNYA_PDF_Z_SHABLONU.md). Його варто враховувати при кожній генерації PDF для друку.

## Блок «посада, місце роботи»

1. У скрипті текст набирається в одному полі з **переносом по словах** (без розриву слів), **по центру**, **жирний**, кегль за замовчуванням **12 pt** — див. розділ «Поле „посада, місце роботи“ у скрипті» нижче.
2. **Збільшення відстані** між рядком `79005, м. Львів` і заголовком `ПОСВІДЧЕННЯ ПРО ВІДРЯДЖЕННЯ` (більше «повітря» зверху) — це зміна **самого шаблону** (Word/PDF верстка). Автоматична підстановка лише заповнює вже наявні поля; глобальний зсув шапки потребує окремого редагування бланку.

## Стандартний випадок (більшість осіб)

- На одному аркуші зазвичай **два однакові примірники** одного посвідчення (дубль для друку/різання) — як у базовому шаблоні.

## Порядок аркушів у зібраному PDF

- **Супровід** (два ПІБ на одному аркуші шаблону з плейсхолдерами «ДМИТРІВУ» тощо) зазвичай ставлять **другим аркушем** одразу після сторінки «Зворотний бік посвідчення…».
- У `keep_original_page_indices` це відповідає порядку на кшталт: `0` (зворотний бік), `6` (аркуш супроводу з шаблону), далі `1, 2, …` (навчання).

## Поле «посада, місце роботи» у скрипті

- Текст набирається **як у текстовому редакторі**: перенос по словах, **по горизонталі по центру**, **жирний** шрифт, **12 pt** (`position_font_size`), нижній край тексту збігається з **нижньою межею** поля шаблону (не «висить» по центру вертикалі).
- Висота блоку **мінімальна** під текст; при довгому тексті поле може **розширитись вгору** в межах `position_field_y_expand_max_pt`, інакше зменшується кегль до `position_min_font_size`.
- `position_field_inset_pt` — невеликий відступ від лівої/правої межі колонки, щоб зменшити накладання між двома примірниками на аркуші.
- **Перенесення «повітря»** від короткої посади в проміжок між рядком «79005, м. Львів» і заголовком «ПОСВІДЧЕННЯ…» при незмінному PDF-шаблоні зробити надійно важко (потрібне зміщення всієї верстки). Для повного контролю макета: зібрати бланк у **Word** (або згенерувати з нуля в Python) і **експортувати в PDF**.

## Особливий випадок: різні підстави / супровід (приклад: Турчин і Копа)

1. У наказі може бути **інша підстава відрядження** та **інша мета**, ніж у блоку «навчання» (наприклад, окремий абзац «У відрядження» до тієї ж частини з іншим формулюванням підстави).
2. Для таких осіб робити **не парні дублікати** (не два однакові бланки на аркуші), а **по одному посвідченню на особу**.
3. **Двох таких осіб** (наприклад, Турчин і Копа) розміщувати **на одному аркуші поруч**: ліва/перша колонка — перша особа, права/друга — друга, кожна з **своїми** реквізитами підстави.
4. **Мета відрядження** для цього типу випадків (коли в наказі — супровід, а не навчання): **`Супровід особового складу`** (як у тексті наказу; перевіряти формулювання в оригіналі).

Реалізація в скрипті (`scripts/vidryadzhennya_pdf_subset_pages.py`):

- У записі `per_page` для сторінки шаблону з **двома однаковими плейсхолдерами ПІБ** (наприклад, обидва «ДМИТРІВУ») задати **`name_new_with_spaces`** і **`position_new_one_line`** як **масиви з двох рядків** — перший елемент для лівої колонки, другий для правої; заміна ПІБ виконується **послідовно** (ліва, потім права).
- Для заміни мети лише на цій сторінці (наприклад «Навчання» → «Супровід особового складу») використати **`page_local_text_pairs`**: список пар `[old, new]` у тому ж форматі, що й глобальні пари (рядок `old` має збігатися з текстом у span).
- Якщо в `page_local_text_pairs` **одна й та сама** підрядок `old` має дати **різні** `new` для лівої та правої колонки, скрипт автоматично застосує **послідовні** заміни (дублікати `old` у списку).
- Опційно **`position_font_size`**, **`position_min_font_size`**, **`position_field_y_expand_max_pt`** — кегль поля посади та запас по вертикалі, якщо текст не вміщується.
- Опційно **`redact_expand_pt`** (за замовчуванням `0.35`) — невелике розширення прямокутників редукції, щоб зменшити обрізання гліфів при друці.

Приклади в локальних `*.local.json` (не в git).

## Безпека

Фактичні ПІБ, номери наказів і шляхи до файлів зберігаються лише в **локальних** JSON поза репозиторієм; у git — лише ці правила та технічний код.
